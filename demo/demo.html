How this file works — your UI for the agent
  1. Check tasks off as they complete; add notes in the ✎ fields. Everything saves to this browser (localStorage).
  2. Hit COPY STATUS FOR AGENT (bottom bar) and paste the block into your agent session (Claude Code, Codex, or any harness running this skill) — that's how the agent reads your state and picks up the next task.
  3. Each task has a ⌘ copy prompt chip: a ready-to-paste instruction for that specific task.
  4. The agent updates this file as work lands. Refreshing never loses your checkmarks or notes — they live in the browser, not the file.

§3

Agent protocol

  • Status sync: COPY STATUS FOR AGENT → paste into any agent session. The agent reconciles, does the next agent-owned tasks, and ships an updated revision of this file.
  • Single task: use a task's ⌘ chip — the prompt carries paths and context so a fresh session needs nothing else.
  • Standing rails: the agent never changes DNS, never wipes the old tower, and never rotates Vaultwarden secrets without the matching go-task checked.
  • File home: ~/projects/homelab-reboot/index.html · memory: homelab-reboot.
